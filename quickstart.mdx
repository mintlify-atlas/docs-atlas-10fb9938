# Quickstart

Get AgroVets App running on your development machine in just a few minutes. This guide assumes you've already completed the [Installation](/installation) steps.

## Start the Development Server

<Steps>
  <Step title="Navigate to Project Directory">
    Open your terminal and navigate to the AgroVets App directory:
    
    ```bash
    cd agrovetsapp
    ```
  </Step>
  
  <Step title="Start Expo Development Server">
    Start the Expo development server:
    
    ```bash
    npm start
    ```
    
    This will start Metro bundler and display a QR code in your terminal.
    
    <Tip>
    The development server will open the Expo DevTools in your browser, showing options to run on different platforms.
    </Tip>
  </Step>
  
  <Step title="Choose Your Platform">
    You can run the app on multiple platforms:
    
    <CodeGroup>
    ```bash Android
    npm run android
    ```
    
    ```bash iOS (macOS only)
    npm run ios
    ```
    
    ```bash Web
    npm run web
    ```
    
    ```bash Expo Go App
    # Scan the QR code with:
    # - Android: Expo Go app
    # - iOS: Camera app
    ```
    </CodeGroup>
    
    <Note>
    For Android and iOS, ensure you have a connected device or emulator running.
    </Note>
  </Step>
</Steps>

## Available Commands

Here are the main npm scripts available in the project:

| Command | Description |
|---------|-------------|
| `npm start` | Start Expo development server (Metro bundler) |
| `npm run android` | Run on Android device/emulator |
| `npm run ios` | Run on iOS device/simulator (macOS with Xcode required) |
| `npm run web` | Run in web browser |

## Explore the App Structure

Once the app is running, you'll see the default navigation structure:

### App Entry Point

The app starts in `App.tsx`, which mounts the main navigation:

```tsx App.tsx
import React from 'react';
import { StatusBar } from 'expo-status-bar';
import AppNavigator from './src/navigation/AppNavigator';

export default function App() {
  return (
    <>
      <AppNavigator />
      <StatusBar style="auto" />
    </>
  );
}
```

### Navigation Flow

The navigation is organized in a hierarchical structure:

<Steps>
  <Step title="AppNavigator (Root)">
    Wraps the entire app with `NavigationContainer`:
    
    ```tsx src/navigation/AppNavigator.tsx
    import React from 'react';
    import { NavigationContainer } from '@react-navigation/native';
    import StackNavigator from './StackNavigator';
    
    export default function AppNavigator() {
      return (
        <NavigationContainer>
          <StackNavigator />
        </NavigationContainer>
      );
    }
    ```
  </Step>
  
  <Step title="StackNavigator">
    Provides stack-based navigation:
    
    ```tsx src/navigation/StackNavigator.tsx
    import React from 'react';
    import { createNativeStackNavigator } from '@react-navigation/native-stack';
    import TabNavigator from './TabNavigator';
    
    const Stack = createNativeStackNavigator();
    
    export default function StackNavigator() {
      return (
        <Stack.Navigator screenOptions={{ headerShown: false }}>
          <Stack.Screen name="Home" component={TabNavigator} />
        </Stack.Navigator>
      );
    }
    ```
  </Step>
  
  <Step title="TabNavigator">
    Displays bottom tab navigation with three screens:
    
    ```tsx src/navigation/TabNavigator.tsx
    import React from 'react';
    import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
    import HomeScreen from '../screens/Home/HomeScreen';
    import ProfileScreen from '../screens/Profile/ProfileScreen';
    import SettingsScreen from '../screens/Settings/SettingsScreen';
    
    const Tab = createBottomTabNavigator();
    
    export default function TabNavigator() {
      return (
        <Tab.Navigator screenOptions={{ headerShown: false }}>
          <Tab.Screen name="Home" component={HomeScreen} />
          <Tab.Screen name="Profile" component={ProfileScreen} />
          <Tab.Screen name="Settings" component={SettingsScreen} />
        </Tab.Navigator>
      );
    }
    ```
  </Step>
</Steps>

## Development Tips

<Tip>
**Live Reloading**: The app supports hot reloading. Save your files and see changes instantly without restarting the app.
</Tip>

<Warning>
If you see errors about missing modules after adding new dependencies, restart the development server with the cache cleared:

```bash
npx expo start -c
```
</Warning>

## TypeScript Support

The project includes full TypeScript support with type definitions for navigation:

```typescript src/navigation/types.ts
export type RootStackParamList = {
  Home: undefined;
  Profile: undefined;
  Settings: undefined;
};
```

Extend this type definition when adding new screens to maintain type safety.

## Project Folders

As you build your app, use these organized folders:

| Folder | Purpose |
|--------|----------|
| `src/screens/` | Screen components for different app views |
| `src/navigation/` | Navigation configuration and navigators |
| `src/components/ui/` | Reusable UI components |
| `src/components/common/` | Common shared components |
| `src/hooks/` | Custom React hooks |
| `src/contexts/` | React contexts for state management |
| `src/services/` | API clients and external service integrations |
| `src/utils/` | Utility functions and helpers |
| `src/theme/` | Styling, colors, and theme configuration |

## Getting Help

If you run into issues:

1. Check the [Installation](/installation) guide for troubleshooting tips
2. Clear Metro cache: `npx expo start -c`
3. Reinstall dependencies: `rm -rf node_modules && npm install`
4. Check Expo documentation: [docs.expo.dev](https://docs.expo.dev)
