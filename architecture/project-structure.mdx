## Overview

AgroVetsApp follows a feature-based folder structure that promotes scalability, maintainability, and clear separation of concerns. This document provides a comprehensive guide to the project's organization.

## Root Directory Structure

```
AgroVetsApp/
├── assets/                 # Static assets (images, fonts, icons)
│   ├── icon.png
│   ├── splash-icon.png
│   ├── adaptive-icon.png
│   └── favicon.png
├── src/                    # Source code directory
│   ├── components/         # Reusable components
│   ├── contexts/           # React Context providers
│   ├── hooks/              # Custom React hooks
│   ├── navigation/         # Navigation configuration
│   ├── screens/            # Screen components
│   ├── services/           # API and business logic
│   ├── theme/              # Styling and theming
│   └── utils/              # Utility functions
├── App.tsx                 # Application entry point
├── index.ts                # Expo entry file
├── app.json                # Expo configuration
├── package.json            # Dependencies and scripts
└── tsconfig.json           # TypeScript configuration
```

## Root Files

### Application Entry Point

<Accordion title="App.tsx">
  **Path**: `App.tsx`
  
  **Lines**: 13
  
  The main entry point for the React Native application.
  
  ```typescript App.tsx
  import React from 'react';
  import { StatusBar } from 'expo-status-bar';
  import AppNavigator from './src/navigation/AppNavigator';
  
  export default function App() {
    return (
      <>
        <AppNavigator />
        <StatusBar style="auto" />
      </>
    );
  }
  ```
  
  **Responsibilities**:
  - Initialize the navigation system
  - Set up global providers (future)
  - Configure status bar
</Accordion>

<Accordion title="index.ts">
  **Path**: `index.ts`
  
  Expo's entry file that registers the root component.
  
  ```typescript index.ts
  import { registerRootComponent } from 'expo';
  import App from './App';
  
  registerRootComponent(App);
  ```
</Accordion>

    "name": "agrovetsapp",
    "version": "1.0.0",

## File Naming Conventions

### Components

- **Format**: PascalCase
- **Extension**: `.tsx`
- **Example**: `Button.tsx`, `HomeScreen.tsx`

### Services & Utilities

- **Format**: camelCase
- **Extension**: `.ts`
- **Example**: `api.ts`, `formatters.ts`

### Types & Interfaces

- **Format**: camelCase for files, PascalCase for types
- **Extension**: `.ts`
- **Example**: `types.ts` (file), `RootStackParamList` (type)

### Index Files

- **Purpose**: Barrel exports for cleaner imports
- **Name**: `index.ts`
- **Example**:

```typescript
// src/components/ui/index.ts
export { default as Button } from './Button';
export { default as Card } from './Card';
```

## Import Path Structure

### Absolute vs Relative Imports

<CodeGroup>

```typescript Relative Imports (Current)
import AppNavigator from './src/navigation/AppNavigator';
import HomeScreen from '../screens/Home/HomeScreen';
```

```typescript Absolute Imports (Recommended)
// Requires path aliases in tsconfig.json
import AppNavigator from '@navigation/AppNavigator';
import HomeScreen from '@screens/Home/HomeScreen';
```

</CodeGroup>

<Note>
  Consider configuring path aliases for cleaner imports as the project grows.
</Note>
